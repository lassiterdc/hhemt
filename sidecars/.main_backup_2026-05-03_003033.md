### Bulletproof plot/table provenance system (full plan-implementation prompt for another session)

> **Cold-start preservation**: this entry is structured as a self-contained plan-implementation seed. A fresh agent invoking `/plan-implementation` with this entry as scope must be able to plan and execute without re-reading the session that surfaced the requirement. Disk-authority rule applies — the file paths and current-state references below are anchors to authority on disk, not session memory.

**Problem this addresses**

The TRITON-SWMM_toolkit's report renderers each consume one or more data sources (zarr stores, NetCDF files, SWMM `.rpt` files, `.inp` files, JSON model logs, GIS shapefiles). For each rendered figure, the toolkit emits a manifest sidecar (`<figure>.manifest.json`) plus a caption RST block ("**Sources:**") that enumerates the sources read. The current implementation declares those sources by hand in collector functions (`_figure_emission.collect_per_sim_source_paths`, `collect_sensitivity_source_paths`, `workflow.py::_collect_per_analysis_summary_source_paths`, `workflow.py::_build_plot_rule_block_system_overview` inline). The hand-declared lists are **provenance enumeration without verification** — there is no runtime check that the renderer actually reads the declared sources, no check that the renderer doesn't read additional undeclared sources, and no check that the declared per-source variable lists match what the renderer actually accesses.

This means the system permits silent provenance drift: a renderer maintainer can add a new variable read or a new file open, the collector list goes stale, and the report's caption claims a complete source set when it isn't. For a system whose intended use case is "scroll the report alongside a notebook to cross-check data without guessing data sources" — and one the user has framed as a national-security imperative where embarrassment scales with single-figure errors in a 1,000-page report — this is unacceptable. The system needs to fail-fast when a renderer's declared source set doesn't match its actual reads.

**Goal**

Make the figure-and-table provenance system bulletproof: every renderer's source declarations are mechanically verified against its actual reads at render time. Mismatches (declared-but-unused; used-but-undeclared; declared variable not actually accessed; actual variable not declared) raise hard exceptions that fail the render. Defense-in-depth via a regression test suite that asserts the cross-check passes for every renderer against the synth fixture. Optional belt-and-suspenders via an AST scan of renderer modules that catches conditional-branch reads the runtime check might miss when the test fixture doesn't trigger the conditional.

**Critical files (anchors on disk; re-read for authority)**

- `src/TRITON_SWMM_toolkit/report_renderers/_figure_emission.py` — `emit_plot_with_sources` is the helper every renderer calls to write the figure + manifest sidecar. The cross-check between declared `source_paths` and actually-read sources lives here.
- `src/TRITON_SWMM_toolkit/report_renderers/_provenance.py` — `ProvenanceLog`, `ProvenanceRef`, `add_channel` infrastructure. Each renderer ALREADY uses `prov.artist().add_channel(ProvenanceRef(source_path=..., variable=...))` for some reads but not uniformly. The bulletproof design promotes `add_channel` from advisory to mandatory and uses its records as the cross-check ground truth.

**Architectural decisions to surface for user approval before code lands**

1. **Where does the cross-check live?** Options: (a) inside `emit_plot_with_sources` after the renderer returns; (b) as a context manager wrapping the renderer's render() call; (c) both. Trade-off: (a) is simplest but only catches mismatches at the closure of render; (b) lets us intercept individual reads as they happen but is more invasive.
2. **What is the failure mode?** Hard `RuntimeError` (recommended for bulletproof framing) vs. warning + continue (more permissive). Recommendation: hard exception, with an opt-out env var for development workflows that need to render before fully wiring provenance.
3. **How strict is variable matching?** Exact string match (e.g., declared `"max_wlevel_m"` must equal actual `ProvenanceRef.variable == "max_wlevel_m"`) vs. fuzzy (substring / canonical-form match). Recommendation: exact match with a normalization helper (strip whitespace, lowercase comparison) — strict enough to catch typos, flexible enough to handle the renderer naming a variable as "max_wlevel_m (peak flood depth raster)".
4. **Conditional reads handling.** Some renderers read sources conditionally (e.g., `per_sim_peak_flood_depth` reads weather NetCDF only in certain panel paths; `sensitivity_benchmarking` reads `.rpt` files only for SWMM-only sub-analyses). The cross-check must distinguish "declared but conditionally not read" from "declared and not read because of bug". Recommendation: add an optional `conditional: bool = False` flag to each source descriptor; cross-check skips the "declared-but-unused" assertion for items with `conditional=True`.
5. **Source descriptor schema.** Currently `dict` with `{"path": str, "variables": list[str]}`. Promote to a Pydantic model (`SourcePathDescriptor`) for type-safety + auto-validation? Recommendation: yes, with optional `conditional`, `format` (zarr/netcdf/csv/rst/json/inp/shapefile), and `description` fields.
6. **Test-suite enforcement scope.** One regression test per renderer against the synth fixture, OR one parameterized test that iterates all renderers? Recommendation: parameterized — easier to add new renderers later.
7. **AST-scan defense layer.** Scan each renderer module for `xr.open_*`, `pd.read_*`, `gpd.read_*`, `swmmio.Model(`, `rxr.open_rasterio`, `open(`, `.read_text()`, etc. Cross-check the literal paths / config-attribute references against the collector's declared paths. Catches conditional-branch reads the runtime check might miss. Trade-off: AST scan is fragile to refactors. Recommendation: ship as a lint check, not a hard gate.

**Implementation sketch (subject to design pass — do not implement without re-reading critical files)**

Phase 1 — foundation:

- Author `SourcePathDescriptor` Pydantic model in a new module `src/TRITON_SWMM_toolkit/report_renderers/_source_descriptor.py` with fields: `path: str`, `variables: list[str]`, `conditional: bool = False`, `format: Literal["zarr", "netcdf", "csv", "rst", "json", "inp", "shapefile", "tif", "geojson", "rpt", "log"]`, `description: str | None`.
